# Variables et types de données

# Étape 2 — Variables et types de données

## 1. La théorie complète

### Déclaration de variables

En PHP, une variable commence **toujours** par `$` suivi d'un nom :

```php
$prenom = "Kevin";   // string
$age = 34;           // int
$taille = 1.80;      // float
$actif = true;       // bool
$rien = null;        // null
```

**Règles de nommage :**

- Commence par `$` + une lettre ou `_` (jamais un chiffre)
- Sensible à la casse : `$nom` ≠ `$Nom`
- Convention professionnelle : **camelCase** → `$nomUtilisateur`, `$dateNaissance`

### Les types dynamiques

PHP est un langage à **typage dynamique** : le type est déterminé automatiquement selon la valeur assignée. Pas besoin de déclarer le type à l'avance.

```php
$valeur = 42;        // int
$valeur = "bonjour"; // devient string — PHP s'adapte
$valeur = true;      // devient bool
```

Tu peux vérifier le type avec `gettype()` et voir le contenu + type avec `var_dump()` :

```php
$age = 34;
echo gettype($age);   // "integer"
var_dump($age);       // int(34)
```

### **Les types scalaires**

```php
// INT — nombre entier
$age = 34;
$annee = 2026;
$temperature = -5;

// FLOAT — nombre décimal
$prix = 19.99;
$tva = 0.21;

// STRING — chaîne de caractères
$prenom = "Kevin";
$message = 'Bonjour le monde';

// BOOL — vrai ou faux
$estConnecte = true;
$estAdmin = false;
```

### **Les types composés**

```php
// ARRAY — tableau (on y reviendra en détail)
$couleurs = ["rouge", "vert", "bleu"];

// OBJECT — objet (on verra ça en POO)
```

### **Les types spéciaux**

```php
// NULL — absence de valeur
$resultat = null;

// RESOURCE — référence vers une ressource externe (fichier, connexion DB)
$fichier = fopen("data.txt", "r"); // type resource
```

### **Variables locales vs globales**

```php
$nom = "Kevin"; // Variable GLOBALE — accessible dans tout le fichier

function direBonjour() {
    // $nom n'est PAS accessible ici directement
    // En PHP, les fonctions ont leur propre portée
    echo $nom; // ❌ Ne fonctionnera pas
}

function direBonjour() {
    global $nom; // ✅ On importe la variable globale
    echo $nom;
}
```

### Constantes

Une constante est une valeur **qui ne change jamais** durant l'exécution :

```php
// Méthode moderne recommandée
const TVA = 0.21;
const NOM_SITE = "MonAgence";

// Ancienne méthode (toujours valide)
define('VERSION', '1.0.0');

// Utilisation — pas de $ pour les constantes
echo TVA;        // 0.21
echo NOM_SITE;   // MonAgence
```

**Différence `const` vs `define()` :**

|  | `const` | `define()` |
| --- | --- | --- |
| Défini à | La compilation | L'exécution |
| Dans une fonction | ❌ Non | ✅ Oui |
| Valeur dynamique | ❌ Non | ✅ Oui |
| Usage moderne | ✅ Préféré | Pour cas spéciaux |

### Les superglobales (aperçu)

Ce sont des variables **automatiquement disponibles partout**, même dans les fonctions :

```php
$_GET       // Données passées dans l'URL
$_POST      // Données envoyées par formulaire
$_SESSION   // Données de session utilisateur
$_COOKIE    // Cookies
$_SERVER    // Infos sur le serveur et la requête
$_FILES     // Fichiers uploadés
$_ENV       // Variables d'environnement
```

On les étudiera en détail quand on arrivera aux formulaires et sessions.

## 2. Exemple concret et utile

Voici une fiche produit typique d'une boutique en ligne :

```php
<?php
/**
 * fiche-produit.php
 * Affiche les informations d'un produit
 */

// Constantes globales du site
const TVA = 0.21;
const NOM_SITE = 'BoutiqueWeb';
const DEVISE = '€';

// Données du produit
$nomProduit     = 'Laptop Pro 15';
$prixHtva       = 899.00;
$stock          = 12;
$estDisponible  = true;
$description    = 'Ordinateur portable haute performance';

// Calculs
$montantTva  = $prixHtva * TVA;
$prixTtc     = $prixHtva + $montantTva;

// Vérification du type
var_dump($estDisponible); // bool(true)
var_dump($stock);         // int(12)
var_dump($prixTtc);       // float(1087.79)

// Affichage
echo "<h1>{$nomProduit}</h1>";
echo "<p>Prix HTVA : {$prixHtva} " . DEVISE . "</p>";
echo "<p>TVA (21%) : {$montantTva} " . DEVISE . "</p>";
echo "<p>Prix TTC : {$prixTtc} " . DEVISE . "</p>";
echo "<p>Stock : {$stock} unités</p>";
```

### **3. Bonnes pratiques et erreurs à éviter**

```php
// ❌ Nommage flou
$x = 34;
$d = "2026-01-01";

// ✅ Nommage expressif
$ageUtilisateur = 34;
$dateInscription = "2026-01-01";

// ❌ Constante en minuscules
const tva = 0.21;

// ✅ Constantes en MAJUSCULES_SNAKE_CASE
const TAUX_TVA = 0.21;

// ❌ Modifier une constante (impossible mais tentant)
const MAX_TENTATIVES = 3;
MAX_TENTATIVES = 5; // Erreur fatale

// ❌ Compter sur le typage dynamique sans le contrôler
$age = "34";     // string !
$result = $age + 1; // PHP convertit... mais c'est dangereux
```

### **4. Ce que font les seniors 👨‍💻**

Les seniors utilisent la **déclaration stricte des types** en haut de chaque fichier :

```php
<?php
declare(strict_types=1); // ← Active le mode strict

/**
 * Avec strict_types, PHP n'accepte plus les conversions implicites
 * Cela évite des bugs silencieux difficiles à détecter
 */

const TAUX_TVA   = 0.21;
const NOM_SITE   = 'BoutiqueWeb';
const VERSION    = '2.1.0';
```

Et ils regroupent toutes les constantes dans un fichier dédié :
```
config/
└── constants.php   ← toutes les constantes du projet
```

```php
<?php
// config/constants.php
declare(strict_types=1);

const TAUX_TVA      = 0.21;
const NOM_SITE      = 'BoutiqueWeb';
const VERSION       = '2.1.0';
const EMAIL_CONTACT = 'contact@boutiqueweb.be';
const MAX_UPLOAD_MB = 5;
```

## 🎯 Exercice 2

Tu travailles pour une agence web. On te demande de créer une **fiche employé** en PHP.

Crée un fichier `employe.php` qui :

1. Déclare `strict_types=1`
2. Crée une constante `NOM_ENTREPRISE` avec le nom de ton choix
3. Déclare ces variables avec les **bons types** :
    - Prénom et nom de l'employé
    - Son âge
    - Son salaire mensuel brut (nombre décimal)
    - S'il est en CDI ou non (booléen)
    - Son département (ex: `"Développement"`)
4. Calcule son **salaire annuel brut**
5. Affiche une page HTML propre avec toutes ces informations dans un tableau HTML `<table>`
6. Utilise `var_dump()` pour vérifier les types de **3 variables** avant l'affichage HTML

Correction :

```php

$salaireAnnuelBrut = $salaireMensuelBrut * 12;

// Vérification des types avant affichage
var_dump($salaireMensuelBrut);  // float(3234.45)

?>

                <th scope="col">Département</th>
```
