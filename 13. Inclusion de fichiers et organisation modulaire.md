# Inclusion de fichiers et organisation modulaire

## 1. La théorie complète

### Les 4 fonctions d'inclusion

```php
// include — inclut le fichier, affiche un WARNING si introuvable et continue
include 'header.php';

// require — inclut le fichier, affiche une ERREUR FATALE si introuvable et stoppe
require 'config.php';

// include_once — comme include mais n'inclut qu'UNE seule fois
include_once 'functions.php';

// require_once — comme require mais n'inclut qu'UNE seule fois
require_once 'functions.php';
```

**La règle simple :**

| Fonction | Si fichier manquant | Inclut plusieurs fois |
| --- | --- | --- |
| `include` | Warning — continue | ✅ Oui |
| `require` | Erreur fatale — stoppe | ✅ Oui |
| `include_once` | Warning — continue | ❌ Non |
| `require_once` | Erreur fatale — stoppe | ❌ Non |

**En pratique :**

```php
// require_once pour tout — c'est le plus sûr
// Si un fichier manque → on veut savoir immédiatement
// _once évite les doublons de fonctions/constantes

require_once 'config/config.php';     // config — indispensable
require_once 'helpers/functions.php'; // fonctions — indispensable
include_once 'templates/header.php';  // template — moins critique
```

### **Chemins absolus vs relatifs**

```php
// ❌ Chemin relatif — dépend d'où le script est appelé
require_once 'config/config.php';
// Peut casser si le fichier est appelé depuis un autre dossier

// ✅ Chemin absolu avec __DIR__
// __DIR__ = répertoire absolu du fichier courant
require_once __DIR__ . '/config/config.php';

// Exemples
echo __DIR__;
// /var/www/html/monprojet

require_once __DIR__ . '/../helpers/functions.php';
// remonte d'un niveau avec ..
```

### Structure professionnelle d'un projet PHP
```
monprojet/
│
├── public/                  ← Seul dossier accessible depuis le web
│   ├── index.php            ← Point d'entrée unique
│   ├── css/
│   │   └── style.css
│   └── js/
│       └── app.js
│
├── src/                     ← Code PHP — non accessible depuis le web
│   ├── config/
│   │   └── config.php       ← Configuration (DB, constantes...)
│   ├── helpers/
│   │   ├── functions.php    ← Fonctions utilitaires
│   │   └── session.php      ← Helpers de session
│   └── templates/
│       ├── header.php       ← En-tête HTML réutilisable
│       ├── footer.php       ← Pied de page réutilisable
│       └── nav.php          ← Navigation
│
└── pages/                   ← Contenu des pages
    ├── accueil.php
    ├── contact.php
    └── articles.php
```

### Le pattern "point d'entrée unique"

Toutes les requêtes passent par `index.php` :

```php
<?php
// public/index.php
declare(strict_types=1);

// Chargement de la configuration
require_once __DIR__ . '/../src/config/config.php';

// Chargement des helpers
require_once __DIR__ . '/../src/helpers/functions.php';
require_once __DIR__ . '/../src/helpers/session.php';

// Démarrage de la session
session_start();

// Routage simple basé sur $_GET['page']
$page = $_GET['page'] ?? 'accueil';

// Liste blanche des pages autorisées
$pagesAutorisees = ['accueil', 'contact', 'articles', 'login', 'dashboard'];

if (!in_array($page, $pagesAutorisees, true)) {
    $page = '404';
}

// Chargement du template
require_once __DIR__ . '/../src/templates/header.php';
require_once __DIR__ . '/../pages/' . $page . '.php';
require_once __DIR__ . '/../src/templates/footer.php';
```

## **3. Bonnes pratiques et erreurs à éviter**

```php
// ❌ Inclure avec un chemin relatif fragile
require_once 'config.php';

// ✅ Toujours utiliser __DIR__
require_once __DIR__ . '/config.php';

// ❌ Mettre la logique métier dans les templates
// header.php
$users = fetchAllUsers();   // ← logique dans un template !

// ✅ Logique dans les pages, templates = affichage uniquement
// page.php
$users = fetchAllUsers();
require_once 'templates/header.php';  // header ne fait qu'afficher

// ❌ Laisser les fichiers de config accessibles depuis le web
// config.php à la racine publique → url.com/config.php visible !

// ✅ Mettre les fichiers sensibles hors du dossier public
// public/  ← accessible depuis le web
// src/     ← non accessible

// ❌ Dupliquer les fonctions dans plusieurs fichiers
// page1.php : function formaterDate() { ... }
// page2.php : function formaterDate() { ... }  ← doublon !

// ✅ Centraliser dans helpers/functions.php
// et inclure avec require_once (le _once empêche les doublons)
require_once __DIR__ . '/helpers/functions.php';
```

## 4. Ce que font les seniors 👨‍💻

Les seniors utilisent un **autoloader** ou un **routeur** mais en procédural simple, ils suivent ce pattern :

```php
<?php
// public/index.php — routeur minimaliste professionnel
declare(strict_types=1);

// Chargement bootstrap
require_once __DIR__ . '/../src/config/config.php';
require_once __DIR__ . '/../src/helpers/functions.php';
require_once __DIR__ . '/../src/helpers/session.php';

session_start();

// Routage
$page   = preg_replace('/[^a-z0-9_-]/', '', $_GET['page'] ?? 'accueil');
$chemin = __DIR__ . '/../pages/' . $page . '.php';

// Vérification — le fichier existe vraiment ?
if (!file_exists($chemin)) {
    $chemin = __DIR__ . '/../pages/404.php';
    http_response_code(404);
}

// Rendu
require_once __DIR__ . '/../src/templates/header.php';
require_once $chemin;
require_once __DIR__ . '/../src/templates/footer.php';
```

Le `preg_replace` nettoie le paramètre `page` pour n'autoriser que des caractères sûrs — évite les attaques par traversée de répertoire (`?page=../../etc/passwd`).

## 🎯 Exercice 10

Tu vas créer un **mini site d'agence web** structuré de manière professionnelle.

### Structure à créer
```
agence/
├── public/
│   └── index.php            ← Point d'entrée + routeur
├── src/
│   ├── config/
│   │   └── config.php       ← Constantes et configuration
│   ├── helpers/
│   │   └── functions.php    ← Fonctions réutilisables
│   └── templates/
│       ├── header.php       ← En-tête HTML
│       ├── nav.php          ← Navigation avec lien actif
│       └── footer.php       ← Pied de page
└── pages/
    ├── accueil.php          ← Page d'accueil
    ├── equipe.php           ← Page équipe
    └── 404.php              ← Page d'erreur
```

### Ce que chaque fichier doit contenir

**`config.php`** :

- `APP_NAME`, `APP_VERSION`, `APP_URL`
- `ANNEE_FONDATION`
- Configuration des erreurs selon l'environnement

**`functions.php`** :

- `formaterDate(string $date): string` — formate une date `Y-m-d` en `"12 mars 2026"`
- `tronquer(string $texte, int $longueur = 100): string` — tronque avec `"..."`
- `genererSlug(string $titre): string` — génère un slug propre

**`header.php`** :

- Utilise `$titrePage` définie dans chaque page
- Inclut `nav.php`

**`nav.php`** :

- Liens vers `accueil`, `equipe`
- Lien actif mis en évidence avec `class="active"`

**`footer.php`** :

- Affiche `APP_NAME`, `APP_VERSION`
- Année dynamique avec `date('Y')`

**`accueil.php`** :

- Définit `$titrePage = "Accueil"`
- Affiche une liste de 3 services avec `foreach`
- Utilise `tronquer()` sur la description de chaque service

**`equipe.php`** :

- Définit `$titrePage = "Notre équipe"`
- Affiche 3 membres avec nom, rôle et date d'arrivée
- Utilise `formaterDate()` sur la date d'arrivée

**`index.php`** :

- Charge `config.php` et `functions.php`
- Route vers la bonne page avec liste blanche
- Affiche la page `404.php` si page inconnue
- Entoure chaque page de `header.php` et `footer.php`

### **`src/helpers/functions.php`**

```php
<?php
declare(strict_types=1);

/**
 * Formate une date Y-m-d en format français lisible
 *
 * @param string $date Date au format Y-m-d (ex: "2021-04-21")
 * @return string      Date formatée (ex: "21 avril 2021")
 */
function formaterDate(string $date): string
{
    $dateObjet = date_create($date);

    if ($dateObjet === false) {
        return 'Date invalide';
    }

    $jour  = date_format($dateObjet, 'd');
    $mois  = (int)date_format($dateObjet, 'm');
    $annee = date_format($dateObjet, 'Y');

    $moisFr = [
        1  => 'janvier',  2  => 'février',   3  => 'mars',
        4  => 'avril',    5  => 'mai',        6  => 'juin',
        7  => 'juillet',  8  => 'août',       9  => 'septembre',
        10 => 'octobre',  11 => 'novembre',   12 => 'décembre'
    ];

    return "{$jour} {$moisFr[$mois]} {$annee}";
}

/**
 * Tronque un texte à une longueur donnée
 *
 * @param string $texte    Texte à tronquer
 * @param int    $longueur Longueur maximale
 * @return string          Texte tronqué ou original
 */
function tronquer(string $texte, int $longueur = 100): string
{
    if (strlen($texte) <= $longueur) {
        return $texte;
    }
    return substr($texte, 0, $longueur) . '...';
}

/**
 * Génère un slug à partir d'un titre
 *
 * @param string $titre Titre brut
 * @return string       Slug formaté
 */
function genererSlug(string $titre): string
{
    $slug = strtolower(trim($titre));
    $slug = str_replace(
        ['à','â','é','è','ê','ë','î','ï','ô','ù','û','ü','ç',' '],
        ['a','a','e','e','e','e','i','i','o','u','u','u','c','-'],
        $slug
    );
    return $slug;
}
```
