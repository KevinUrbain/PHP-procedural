# Sessions et cookies

## 1. La théorie complète

### Le problème fondamental — HTTP est sans état

```
HTTP est un protocole SANS ÉTAT (stateless)
Chaque requête est indépendante — le serveur ne se souvient de rien

Page 1 : L'utilisateur se connecte      → Serveur : "OK"
Page 2 : L'utilisateur accède au profil → Serveur : "Qui êtes-vous ?"

Sans sessions/cookies → impossible de savoir qui est connecté
```

### Les sessions — côté serveur

Une session stocke les données **sur le serveur**. Le navigateur ne reçoit qu'un identifiant unique (le `session_id`) dans un cookie.

```
Navigateur                         Serveur
    │                                  │
    │  1. POST /login                  │
    │ ─────────────────────────────►  │
    │                                  │ Crée une session
    │  2. Cookie: PHPSESSID=abc123     │ $_SESSION['user'] = "Jack"
    │ ◄─────────────────────────────  │
    │                                  │
    │  3. GET /profil                  │
    │     Cookie: PHPSESSID=abc123 ──►│
    │                                  │ Retrouve la session via abc123
    │  4. Affiche le profil de Jack   │ Sait que c'est Jack ✅
    │ ◄─────────────────────────────  │
```

### **Démarrer et utiliser une session**

```php
<?php
// session_start() DOIT être la première chose dans le fichier
// Avant tout output HTML, echo, espace...
session_start();

// Stocker des données
$_SESSION['prenom']      = 'Jack';
$_SESSION['role']        = 'admin';
$_SESSION['connecte']    = true;

// Lire des données
echo $_SESSION['prenom'];          // "Jack"

// Vérifier si une clé existe
if (isset($_SESSION['connecte'])) {
    echo "Utilisateur connecté";
}

// Lire avec valeur par défaut
$role = $_SESSION['role'] ?? 'visiteur';

// Modifier
$_SESSION['role'] = 'editor';

// Supprimer une clé
unset($_SESSION['prenom']);

// Supprimer TOUTES les données de session
session_unset();

// Détruire complètement la session (déconnexion)
session_destroy();
```

### Pattern de connexion complet

```php
<?php
// login.php
session_start();

// Si déjà connecté → rediriger
if (isset($_SESSION['connecte']) && $_SESSION['connecte'] === true) {
    header('Location: dashboard.php');
    exit;
}

$erreurs = [];

if ($_SERVER['REQUEST_METHOD'] === 'POST') {

    $email    = trim($_POST['email']    ?? '');
    $password = $_POST['password']      ?? '';

    // Simulation d'un utilisateur en base de données
    // En réalité → requête SQL
    $utilisateurEnDB = [
        'id'       => 1,
        'prenom'   => 'Jack',
        'email'    => 'jack@exemple.com',
        'role'     => 'admin',
        // Mot de passe hashé avec password_hash() — on verra ça en sécurité
        'password' => password_hash('motdepasse123', PASSWORD_BCRYPT)
    ];

    // Vérification email + mot de passe
    if ($email !== $utilisateurEnDB['email']) {
        $erreurs[] = "Email ou mot de passe incorrect";
    } elseif (!password_verify($password, $utilisateurEnDB['password'])) {
        $erreurs[] = "Email ou mot de passe incorrect";
    }

    // Connexion réussie
    if (empty($erreurs)) {
        // Régénérer l'ID de session — sécurité anti-fixation
        session_regenerate_id(true);

        // Stocker les infos en session
        $_SESSION['connecte'] = true;
        $_SESSION['user_id']  = $utilisateurEnDB['id'];
        $_SESSION['prenom']   = $utilisateurEnDB['prenom'];
        $_SESSION['role']     = $utilisateurEnDB['role'];

        header('Location: dashboard.php');
        exit;
    }
}
?>
```

### **Pattern de protection de page**

```php
<?php
// dashboard.php — page protégée
session_start();

// Guard clause — si pas connecté, on redirige immédiatement
if (!isset($_SESSION['connecte']) || $_SESSION['connecte'] !== true) {
    header('Location: login.php');
    exit;
}

// Guard clause — si pas le bon rôle
if ($_SESSION['role'] !== 'admin') {
    header('Location: acces-refuse.php');
    exit;
}

// Ici on est sûr que l'utilisateur est connecté et admin
$prenom = $_SESSION['prenom'];
?>

<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <title>Dashboard</title>
</head>
<body>
    <h1>Bonjour <?= htmlspecialchars($prenom, ENT_QUOTES, 'UTF-8') ?> !</h1>
    <a href="logout.php">Se déconnecter</a>
</body>
</html>
```

### **Pattern de déconnexion**

```php
<?php
// logout.php
session_start();

// 1. Supprimer toutes les données de session
session_unset();

// 2. Détruire la session côté serveur
session_destroy();

// 3. Supprimer le cookie de session côté navigateur
if (isset($_COOKIE[session_name()])) {
    setcookie(
        session_name(),
        '',
        time() - 3600,  // Date dans le passé → supprime le cookie
        '/',
        '',
        true,   // secure — HTTPS seulement en production
        true    // httponly — inaccessible en JavaScript
    );
}

// 4. Rediriger
header('Location: login.php');
exit;
```

### Les cookies — côté navigateur

Un cookie est un petit fichier texte stocké **dans le navigateur** de l'utilisateur.
```
Sessions  → données sur le SERVEUR  → plus sécurisé
Cookies   → données dans le NAVIGATEUR → accessible côté client
```

```php
// Créer un cookie
// setcookie(nom, valeur, expiration, chemin, domaine, secure, httponly)
setcookie(
    'theme',           // Nom
    'dark',            // Valeur
    time() + 86400,    // Expiration : maintenant + 1 jour (en secondes)
    '/',               // Chemin : tout le site
    '',                // Domaine : domaine actuel
    false,             // Secure : true en production (HTTPS uniquement)
    true               // HttpOnly : inaccessible en JavaScript (sécurité)
);

// Lire un cookie
$theme = $_COOKIE['theme'] ?? 'light';  // Valeur par défaut si inexistant

// Modifier un cookie — on le recrée avec les nouvelles valeurs
setcookie('theme', 'light', time() + 86400, '/');

// Supprimer un cookie — date dans le passé
setcookie('theme', '', time() - 3600, '/');
```

### Sessions vs Cookies — quand utiliser quoi

|  | Sessions | Cookies |
| --- | --- | --- |
| Stockage | Serveur | Navigateur |
| Sécurité | ✅ Plus sûr | ⚠️ Modifiable par l'utilisateur |
| Durée | Jusqu'à fermeture navigateur (défaut) | Configurable (jours, mois, années) |
| Taille | Illimitée | 4Ko max |
| Usage | Connexion, panier, données sensibles | Préférences, thème, langue |

### Durées en secondes — référence

```php
time() + 60          // 1 minute
time() + 3600        // 1 heure
time() + 86400       // 1 jour
time() + 604800      // 1 semaine
time() + 2592000     // 30 jours
time() + 31536000    // 1 an
```

## 2. Exemple concret et utile

Un système de **préférences utilisateur** combinant sessions et cookies :

```php
<?php
// preferences.php
declare(strict_types=1);

session_start();

const THEMES_AUTORISES  = ['light', 'dark', 'system'];
const LANGUES_AUTORISEES = ['fr', 'en', 'nl'];

// Traitement du changement de préférences
if ($_SERVER['REQUEST_METHOD'] === 'POST') {

    $theme  = $_POST['theme']  ?? 'light';
    $langue = $_POST['langue'] ?? 'fr';

    // Validation
    if (!in_array($theme, THEMES_AUTORISES, true)) {
        $theme = 'light';
    }
    if (!in_array($langue, LANGUES_AUTORISEES, true)) {
        $langue = 'fr';
    }

    // Stocker en cookie — préférences persistantes (30 jours)
    setcookie('theme',  $theme,  time() + 2592000, '/', '', false, true);
    setcookie('langue', $langue, time() + 2592000, '/', '', false, true);

    // Stocker en session aussi — disponible immédiatement
    $_SESSION['theme']  = $theme;
    $_SESSION['langue'] = $langue;

    header('Location: preferences.php?succes=1');
    exit;
}

// Lire les préférences — session en priorité, cookie en fallback, défaut sinon
$theme  = $_SESSION['theme']  ?? $_COOKIE['theme']  ?? 'light';
$langue = $_SESSION['langue'] ?? $_COOKIE['langue'] ?? 'fr';

$afficherSucces = isset($_GET['succes']) && $_GET['succes'] === '1';
?>
```

## **3. Bonnes pratiques et erreurs à éviter**

```php
// ❌ Oublier session_start() en premier
echo "Bonjour";
session_start();   // Fatal error — output déjà envoyé !

// ✅ session_start() absolument en premier — avant tout output
<?php
session_start();
// ...
?>

// ❌ Stocker des données sensibles en cookie
setcookie('user_id', '42', time() + 86400);
// Un utilisateur peut modifier son cookie et se faire passer pour l'ID 42 !

// ✅ Stocker l'ID en session — jamais en cookie non signé
$_SESSION['user_id'] = 42;

// ❌ Ne pas régénérer l'ID de session après connexion
$_SESSION['connecte'] = true;
// Faille : session fixation attack

// ✅ Toujours régénérer après connexion
session_regenerate_id(true);
$_SESSION['connecte'] = true;

// ❌ Ne jamais stocker le mot de passe — même hashé — en session
$_SESSION['password'] = $hashedPassword;

// ✅ Stocker uniquement ce qui est nécessaire
$_SESSION['user_id'] = $user['id'];
$_SESSION['role']    = $user['role'];
```

## 4. Ce que font les seniors 👨‍💻

Les seniors créent des **fonctions helpers** pour centraliser la gestion de session :

```php
<?php
// helpers/session.php
declare(strict_types=1);

/**
 * Vérifie si l'utilisateur est connecté
 */
function estConnecte(): bool
{
    return isset($_SESSION['connecte']) && $_SESSION['connecte'] === true;
}

/**
 * Vérifie si l'utilisateur a un rôle donné
 */
function aLeRole(string $role): bool
{
    return isset($_SESSION['role']) && $_SESSION['role'] === $role;
}

/**
 * Protège une page — redirige si non connecté
 */
function protegerPage(string $roleRequis = ''): void
{
    if (!estConnecte()) {
        header('Location: /login.php');
        exit;
    }

    if ($roleRequis !== '' && !aLeRole($roleRequis)) {
        header('Location: /acces-refuse.php');
        exit;
    }
}

/**
 * Déconnecte l'utilisateur complètement
 */
function deconnecter(): void
{
    session_unset();
    session_destroy();

    if (isset($_COOKIE[session_name()])) {
        setcookie(session_name(), '', time() - 3600, '/');
    }

    header('Location: /login.php');
    exit;
}
```

### Et dans chaque page protégée — une seule ligne :

```php
<?php
session_start();
require_once 'helpers/session.php';

protegerPage('admin');   // Protège et vérifie le rôle en une ligne
?>
```

## 🎯 Exercice 9

Tu dois créer un **mini système d'authentification** avec 3 fichiers.

### Fichier 1 — `login.php`

- Formulaire avec `email` et `password`
- Valide les champs (non vides, email valide)
- Simule un utilisateur en dur dans un tableau :

```php
$utilisateurs = [
    ['id' => 1, 'prenom' => 'Jack',  'email' => 'jack@test.com',  'role' => 'admin',  'password' => 'Admin1234'],
    ['id' => 2, 'prenom' => 'Alice', 'email' => 'alice@test.com', 'role' => 'editor', 'password' => 'Alice5678'],
];
```

- Si email + password correspondent → connexion + redirection vers `dashboard.php`
- Si déjà connecté → rediriger directement vers `dashboard.php`
- Affiche les erreurs sans préciser si c'est l'email ou le mot de passe qui est faux (sécurité)

### Fichier 2 — `dashboard.php`

- Protège la page — si non connecté → rediriger vers `login.php`
- Affiche : `"Bonjour [prenom], vous êtes connecté en tant que [role]"`
- Affiche un lien vers `logout.php`
- Si le rôle est `admin` → afficher un message `"Panneau administrateur"`
- Si le rôle est `editor` → afficher un message `"Panneau éditeur"`

### Fichier 3 — `logout.php`

- Détruit la session complètement
- Supprime le cookie de session
- Redirige vers `login.php`

### Bonus — cookie de préférence

- Sur `login.php` — une case à cocher `"Se souvenir de moi"`
- Si cochée → stocker l'email dans un cookie 30 jours
- Au prochain chargement de `login.php` → repopuler le champ email depuis le cookie

### **`login.php`**

```php
<?php
declare(strict_types=1);

/**
 * login.php
 * Formulaire de connexion
 */

session_start();

// ✅ Guard clause — si déjà connecté, rediriger immédiatement
if (isset($_SESSION['user']) && $_SESSION['user']['connecte'] === true) {
    header('Location: dashboard.php');
    exit;
}

// Utilisateurs simulés (en réalité → base de données)
$utilisateurs = [
    ['id' => 1, 'prenom' => 'Jack',  'email' => 'jack@test.com',  'role' => 'admin',  'password' => 'Admin1234'],
    ['id' => 2, 'prenom' => 'Alice', 'email' => 'alice@test.com', 'role' => 'editor', 'password' => 'Alice5678'],
];

$erreurs = [];
$email   = '';

// Repopuler l'email depuis le cookie "Se souvenir de moi"
if (isset($_COOKIE['remember_email'])) {
    $email = htmlspecialchars($_COOKIE['remember_email'], ENT_QUOTES, 'UTF-8');
}

if ($_SERVER['REQUEST_METHOD'] === 'POST') {

    $email    = trim($_POST['email']    ?? '');
    $password = $_POST['password']      ?? '';
    $remember = isset($_POST['remember']); // case à cocher

    // --- Validation ---
    if (empty($email)) {
        $erreurs[] = "L'email est obligatoire";
    } elseif (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $erreurs[] = "Le format de l'email n'est pas valide";
    }

    if (empty($password)) {
        $erreurs[] = "Le mot de passe est obligatoire";
    }

    // --- Recherche utilisateur UNIQUEMENT si validation OK ---
    if (empty($erreurs)) {

        $resultats   = array_values(array_filter(
            $utilisateurs,
            fn(array $u): bool => $u['email'] === $email
        ));
        $utilisateur = $resultats[0] ?? null;   // null si non trouvé

        // Message générique — ne pas révéler si c'est l'email ou le MDP qui est faux
        if ($utilisateur === null || $password !== $utilisateur['password']) {
            $erreurs[] = "Email ou mot de passe incorrect";
        }
    }

    // --- Connexion si tout est OK ---
    if (empty($erreurs)) {

        // Régénérer l'ID de session — protection contre la fixation de session
        session_regenerate_id(true);

        // Stocker les infos nécessaires en session
        $_SESSION['user'] = [
            'id'       => $utilisateur['id'],
            'prenom'   => $utilisateur['prenom'],
            'role'     => $utilisateur['role'],
            'connecte' => true
        ];

        // Bonus — "Se souvenir de moi"
        if ($remember) {
            // Stocker l'email en cookie 30 jours
            setcookie('remember_email', $email, time() + 2592000, '/', '', false, true);
        } else {
            // Supprimer le cookie si la case n'est pas cochée
            setcookie('remember_email', '', time() - 3600, '/');
        }

        header('Location: dashboard.php');
        exit;
    }
}
?>
<!DOCTYPE html>
<html lang="fr">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Connexion</title>
</head>
<body>

    <h1>Connexion</h1>

    <?php if (!empty($erreurs)) : ?>
        <ul style="color: red">
            <?php foreach ($erreurs as $erreur) : ?>
                <li><?= htmlspecialchars($erreur, ENT_QUOTES, 'UTF-8') ?></li>
            <?php endforeach; ?>
        </ul>
    <?php endif; ?>

    <form method="POST" action="">

        <div>
            <label for="email">Email *</label><br>
            <input
                type="email"
                id="email"
                name="email"
                value="<?= htmlspecialchars($email, ENT_QUOTES, 'UTF-8') ?>"
            >
        </div>

        <div>
            <label for="password">Mot de passe *</label><br>
            <input type="password" id="password" name="password" value="">
        </div>

        <div>
            <label>
                <input type="checkbox" name="remember">
                Se souvenir de moi
            </label>
        </div>

        <button type="submit">Se connecter</button>

    </form>

</body>
</html>
```

## ⚠️ Point à faire attention :

### ❌ L'ID utilisateur exposé dans l'URL

```php
// ❌ Exposer l'ID en GET — mauvaise pratique de sécurité
header('Location: dashboard.php?succesLogin=1&id=' . $utilisateur['id']);

// L'ID est déjà dans $_SESSION — pas besoin de le mettre dans l'URL
// Un utilisateur pourrait manipuler ?id=3 pour accéder à un autre compte
```

### 🟢 La vérification "déjà connecté" doit être **tout en haut du fichier**, avant tout traitement :

```php
// ✅ En haut du fichier — avant le if POST
session_start();

// Si déjà connecté → rediriger immédiatement
if (isset($_SESSION['user']) && $_SESSION['user']['connecte'] === true) {
    header('Location: dashboard.php');
    exit;
}

// Suite du traitement...
if ($_SERVER['REQUEST_METHOD'] === 'POST') { ... }
```

### 🔧 `htmlspecialchars` incomplet

```php
// ❌ Sans ENT_QUOTES et UTF-8
<?= htmlspecialchars($email) ?>
<?= $_SESSION['user']['prenom'] ?>   // ← pas de htmlspecialchars du tout !

// ✅ Toujours complet
<?= htmlspecialchars($email, ENT_QUOTES, 'UTF-8') ?>
<?= htmlspecialchars($_SESSION['user']['prenom'], ENT_QUOTES, 'UTF-8') ?>
```
