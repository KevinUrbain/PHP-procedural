# Étape 3 — Les opérateurs

## 1. La théorie complète

### Opérateurs arithmétiques

```php
$a = 10;
$b = 3;

echo $a + $b;   // 13  — addition
echo $a - $b;   // 7   — soustraction
echo $a * $b;   // 30  — multiplication
echo $a / $b;   // 3.333... — division
echo $a % $b;   // 1   — modulo (reste de la division)
echo $a ** $b;  // 1000 — puissance (10³)
```

### **Opérateurs d'assignation**

```php
$a = 10;

$a += 5;   // équivaut à $a = $a + 5  → 15
$a -= 3;   // équivaut à $a = $a - 3  → 12
$a *= 2;   // équivaut à $a = $a * 2  → 24
$a /= 4;   // équivaut à $a = $a / 4  → 6
$a %= 4;   // équivaut à $a = $a % 4  → 2

// Concaténation avec assignation
$texte = "Bonjour";
$texte .= " Kevin";  // équivaut à $texte = $texte . " Kevin"
echo $texte;         // "Bonjour Kevin"
```

### Opérateurs de comparaison

C'est ici que beaucoup de débutants font des erreurs critiques :

```php
$a = 5;
$b = "5";

// == égalité LAXISTE — compare la valeur, ignore le type
var_dump($a == $b);   // bool(true)  ← dangereux

// === égalité STRICTE — compare valeur ET type
var_dump($a === $b);  // bool(false) ← correct

// != différence laxiste
var_dump($a != $b);   // bool(false)

// !== différence stricte
var_dump($a !== $b);  // bool(true)

// Comparaisons numériques
var_dump($a > 3);     // bool(true)
var_dump($a < 3);     // bool(false)
var_dump($a >= 5);    // bool(true)
var_dump($a <= 4);    // bool(false)
```

### Opérateur spaceship `<=>`

Retourne `-1`, `0` ou `1` selon que le membre gauche est inférieur, égal ou supérieur :

```php
echo 1 <=> 2;   // -1  (1 est inférieur à 2)
echo 2 <=> 2;   //  0  (égaux)
echo 3 <=> 2;   //  1  (3 est supérieur à 2)
```

Très utilisé pour **trier des tableaux** — on y reviendra.

### **Opérateurs logiques**

```php
$estConnecte = true;
$estAdmin    = false;

// && — ET logique (les deux doivent être vrais)
var_dump($estConnecte && $estAdmin);   // bool(false)

// || — OU logique (au moins un doit être vrai)
var_dump($estConnecte || $estAdmin);   // bool(true)

// ! — NON logique (inverse)
var_dump(!$estConnecte);               // bool(false)
var_dump(!$estAdmin);                  // bool(true)
```

### **Opérateurs d'incrémentation**

```php
$i = 5;

// Pré-incrémentation — incrémente AVANT de retourner la valeur
echo ++$i;   // 6 — $i vaut 6

// Post-incrémentation — retourne la valeur PUIS incrémente
echo $i++;   // 6 — affiche 6, mais $i vaut maintenant 7
echo $i;     // 7

// Même logique pour la décrémentation
echo --$i;   // 6
echo $i--;   // 6 — affiche 6, mais $i vaut maintenant 5
```

### **Opérateur de concaténation**

```php
$prenom = "Kevin";
$nom    = "Jones";

// . pour concaténer
$nomComplet = $prenom . " " . $nom;
echo $nomComplet;   // "Kevin Jones"

// .= pour concaténer et assigner
$message  = "Bonjour ";
$message .= $prenom;
echo $message;      // "Bonjour Kevin"
```

### Opérateur ternaire `? :`

Raccourci d'un `if / else` sur une ligne :

```php
// Syntaxe : condition ? valeur_si_vrai : valeur_si_faux
$age = 20;

$statut = $age >= 18 ? "Majeur" : "Mineur";
echo $statut;   // "Majeur"

// Équivalent long
if ($age >= 18) {
    $statut = "Majeur";
} else {
    $statut = "Mineur";
}
```

### Opérateur null coalescent `??`

Retourne la valeur de gauche si elle **existe et n'est pas null**, sinon retourne la valeur de droite :

```php
// Sans ?? — vérification manuelle lourde
$pseudo = isset($_GET['pseudo']) ? $_GET['pseudo'] : 'Anonyme';

// Avec ?? — clean et moderne
$pseudo = $_GET['pseudo'] ?? 'Anonyme';

// Chaînage possible
$nom = $prenom ?? $pseudo ?? 'Inconnu';
```

C'est **très utilisé** avec les formulaires et les superglobales.

## 2. Exemple concret et utile

Simulons le calcul d'une **facture avec remise** comme on en voit en entreprise :

```php
<?php
declare(strict_types=1);

/**
 * facture.php
 * Calcul d'une facture avec remise et TVA
 */

const TAUX_TVA     = 0.21;
const TAUX_REMISE  = 0.10;  // 10% de remise

// Données de la commande
$nomClient     = 'Jack Jones';
$produit       = 'Laptop Pro 15';
$prixUnitaire  = 899.00;
$quantite      = 3;

// Calculs
$sousTotal     = $prixUnitaire * $quantite;
$montantRemise = $sousTotal * TAUX_REMISE;
$apresRemise   = $sousTotal - $montantRemise;
$montantTva    = $apresRemise * TAUX_TVA;
$totalTtc      = $apresRemise + $montantTva;

// Le client est-il éligible à la remise ? (commande > 1000€)
$remiseAppliquee = $sousTotal > 1000 ? 'Oui' : 'Non';

// Référence de commande — utilise ?? pour valeur par défaut
$reference = $_GET['ref'] ?? 'REF-0000';

// Affichage
echo "<h2>Facture — {$nomClient}</h2>";
echo "<p>Référence : {$reference}</p>";
echo "<p>Produit : {$produit}</p>";
echo "<p>Quantité : {$quantite}</p>";
echo "<p>Sous-total : " . number_format($sousTotal, 2, ',', ' ') . " €</p>";
echo "<p>Remise (10%) appliquée : {$remiseAppliquee}</p>";
echo "<p>Montant remise : " . number_format($montantRemise, 2, ',', ' ') . " €</p>";
echo "<p>TVA (21%) : " . number_format($montantTva, 2, ',', ' ') . " €</p>";
echo "<p><strong>Total TTC : " . number_format($totalTtc, 2, ',', ' ') . " €</strong></p>";
```

### **3. Bonnes pratiques et erreurs à éviter**

```php
// ❌ Toujours utiliser == (comparaison laxiste)
if ($role == "admin") { }    // "admin" == 0 est TRUE en PHP !

// ✅ Toujours utiliser === (comparaison stricte)
if ($role === "admin") { }

// ❌ Ternaire imbriqué illisible
$x = $a ? $b ? 'bb' : 'b' : 'a';

// ✅ Lisible et explicite
if ($a) {
    $x = $b ? 'bb' : 'b';
} else {
    $x = 'a';
}

// ❌ Ignorer le type dans les calculs
$prix = "19.99";
$total = $prix * 2;   // PHP convertit silencieusement — dangereux

// ✅ S'assurer du type
$prix = 19.99;        // float dès le départ
$total = $prix * 2;
```

## 4. Ce que font les seniors 👨‍💻

Les seniors combinent les opérateurs pour écrire une logique **dense mais lisible** :

```php
<?php
declare(strict_types=1);

// Récupération sécurisée depuis un formulaire
$quantite  = (int)($_POST['quantite'] ?? 1);    // cast en int + valeur par défaut
$codePromo = trim($_POST['code_promo'] ?? '');  // nettoyage + valeur par défaut

// Calcul conditionnel en une ligne
$remise    = $codePromo === 'PROMO10' ? 0.10 : 0.00;
$estVip    = $quantite >= 10 && $remise > 0;

// Assignation conditionnelle propre
$messageRemise = $remise > 0
    ? "Remise de " . ($remise * 100) . "% appliquée"
    : "Aucune remise";

echo $messageRemise;
```

Le cast `(int)`, `(float)`, `(string)` permet de **forcer le type** d'une valeur — une protection importante contre les données utilisateur non fiables.
