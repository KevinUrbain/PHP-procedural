# Syntaxe PHP de base

# 🔹 Étape 1 — Syntaxe PHP de base

## 1. La théorie complète

PHP est un langage **côté serveur** : le code est interprété par le serveur, et le navigateur ne reçoit que du HTML en retour.

### Les balises PHP

```php
<?php
    // Ton code ici
?>
```

**Règles fondamentales :**

- Tout fichier PHP porte l'extension `.php`
- Le code PHP vit **entre** `<?php` et `?>`
- Quand le fichier ne contient **que** du PHP (pas de HTML), on **omet la balise fermante** `?>` — c'est une bonne pratique professionnelle pour éviter des espaces parasites qui cassent les headers HTTP
- Chaque instruction se termine par un **point-virgule** `;`

### Les commentaires

```php
<?php

// Commentaire sur une ligne (le plus utilisé)

# Commentaire sur une ligne (style shell, rare en PHP)

/*
   Commentaire
   sur plusieurs lignes
*/

/**
 * PHPDoc — commentaire professionnel pour documenter une fonction
 * Utilisé par les IDE et les outils de documentation
 */
```

### **Sensibilité à la casse**

```php
<?php

// Les variables sont CAS-SENSITIVES
$nom = "Alice";
$Nom = "Bob";   // Variable DIFFÉRENTE de $nom

// Les fonctions et mots-clés ne sont PAS sensibles à la casse
echo "Bonjour";
ECHO "Bonjour";   // Fonctionne aussi (mais à éviter)
```

### **2. Exemple concret et utile**

```php
<?php
// Fichier : index.php
// Rôle : page d'accueil d'un site vitrine

// On définit le nom du site
$nomSite = "MonAgence";

// On affiche le HTML avec PHP
echo "<!DOCTYPE html>";
echo "<html lang='fr'>";
echo "<head><title>" . $nomSite . "</title></head>";
echo "<body>";
echo "<h1>Bienvenue sur " . $nomSite . "</h1>";
echo "</body>";
echo "</html>";
```

## 3. Bonnes pratiques professionnelles ✅

| ✅ À faire | ❌ À éviter |
|---|---|
| Omettre `?>` dans les fichiers 100% PHP | Laisser des espaces après `?>` |
| Indenter avec 4 espaces | Mélanger tabs et espaces |
| Un fichier = un rôle clair | Tout mettre dans un seul fichier |
| Commenter le **pourquoi**, pas le **quoi** | `// On affiche h1` (inutile) |
| Nommer les fichiers en minuscules avec tirets | `MonFichier.PHP` |

## 4. Ce que font les seniors 👨‍💻

Un développeur senior structure dès le départ :
```
projet/
├── index.php          ← point d'entrée
├── config/
│   └── config.php     ← configuration (DB, constantes)
├── includes/
│   ├── header.php     ← en-tête HTML réutilisable
│   └── footer.php     ← pied de page réutilisable
└── pages/
    └── accueil.php    ← contenu de la page
```

Et `index.php` ressemble à ça :

```php
<?php
// Pas de balise fermante ?> — bonne pratique
require_once 'includes/header.php';
require_once 'pages/accueil.php';
require_once 'includes/footer.php';
```

## 🎯 Exercice 1

Crée un fichier `index.php` qui :

1. Contient **uniquement** du PHP (pas de HTML écrit à la main en dehors de PHP)
2. Déclare une variable `$prenom` avec ton prénom
3. Déclare une variable `$metier` avec `"Développeur web"`
4. Affiche avec `echo` une page HTML complète (doctype, html, head, body) avec un `<h1>` qui dit : **"Bonjour, je suis [ton prénom], [ton métier]"**
5. Ajoute **3 types de commentaires** différents dans ton code
6. N'oublie pas : **pas de balise fermante** `?>`

Correction :

```php
<?php
/**
 * index.php
 * Page d'accueil — affiche une présentation personnelle
 * @author Kevin
 */

// Données de présentation
$prenom = 'Kevin';
$metier = 'Développeur web';

/*
    Structure HTML complète de la page
    Les variables sont interpolées avec la syntaxe {$var}
*/

echo '<!DOCTYPE html>';
echo '<html lang="fr">';
echo '<head>';
echo '    <meta charset="UTF-8">';
echo '    <meta name="viewport" content="width=device-width, initial-scale=1.0">';
echo "    <title>Page de {$prenom}</title>";
echo '</head>';
echo '<body>';
echo "    <h1>Bonjour, je suis {$prenom}, {$metier}</h1>";
echo '</body>';
echo '</html>';
```
