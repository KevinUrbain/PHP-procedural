# Les callbacks dans les fonctions de tableaux

# PARTIE 1 — C'est quoi un callback ?

Un callback c'est simplement **une fonction passée en argument à une autre fonction**.

```php
// Fonction normale — on lui passe des valeurs
calculerMoyenne(15.5, 12.3);

// Fonction avec callback — on lui passe une FONCTION
array_map(maFonction, $tableau);
//         ↑
//    C'est le callback
```

L'idée c'est que `array_map` ne sait pas comment transformer tes données — **c'est toi qui lui dis quoi faire** via le callback.

# PARTIE 2 — Les trois façons d'écrire un callback

## 1. Fonction nommée — la plus simple

```php
// Tu déclares une fonction normale
function doubler(int $n): int
{
    return $n * 2;
}

$nombres = [1, 2, 3, 4, 5];

// Tu passes son NOM en string comme callback
$doubles = array_map('doubler', $nombres);
// [2, 4, 6, 8, 10]
```

## **2. Fonction anonyme — la plus utilisée**

```php
$nombres = [1, 2, 3, 4, 5];

// La fonction est définie DIRECTEMENT dans l'appel
$doubles = array_map(function(int $n): int {
    return $n * 2;
}, $nombres);
// [2, 4, 6, 8, 10]
```

La fonction n'a pas de nom — elle existe juste pour cet appel.

## 3. Arrow function — la plus concise (PHP 7.4+)

```php
$nombres = [1, 2, 3, 4, 5];

// Version ultra-courte pour les cas simples
$doubles = array_map(fn(int $n): int => $n * 2, $nombres);
// [2, 4, 6, 8, 10]
```

## Quand utiliser laquelle ?

| Situation | Choix |
| --- | --- |
| Logique réutilisée ailleurs | Fonction nommée |
| Logique simple, une ligne | Arrow function `fn` |
| Logique complexe, plusieurs lignes | Fonction anonyme |

# PARTIE 3 — Ce que reçoit le callback

C'est **crucial** de comprendre ce que chaque fonction native passe à ton callback.

## `array_map` — reçoit chaque valeur

```php
$produits = [
    ["nom" => "Laptop", "prix" => 899.0],
    ["nom" => "Souris", "prix" => 29.0],
];

// array_map passe CHAQUE élément du tableau à ton callback
// $p = ["nom" => "Laptop", "prix" => 899.0] au 1er tour
// $p = ["nom" => "Souris", "prix" => 29.0]  au 2ème tour

$enrichis = array_map(function(array $p): array {
    $p['prixTTC'] = $p['prix'] * 1.21;
    return $p;   // ← OBLIGATOIRE — array_map attend un retour
}, $produits);
```

⚠️ **`array_map` attend toujours un `return`** — si tu oublies, le tableau sera rempli de `null`.

## **`array_filter` — reçoit chaque valeur, attend un bool**

```php
$produits = [
    ["nom" => "Laptop", "stock" => 5],
    ["nom" => "Souris", "stock" => 0],
    ["nom" => "Clavier","stock" => 3],
];

// array_filter passe chaque élément à ton callback
// Si le callback retourne TRUE  → l'élément est GARDÉ
// Si le callback retourne FALSE → l'élément est SUPPRIMÉ

$enStock = array_filter($produits, function(array $p): bool {
    return $p['stock'] > 0;   // true = garder, false = supprimer
});

// Résultat : Laptop et Clavier (Souris supprimée car stock = 0)
```

## `array_reduce` — reçoit l'accumulateur ET la valeur

C'est le plus complexe à comprendre. Voici la mécanique pas à pas :

```php
$notes = [15.0, 12.0, 18.0, 9.0];

// array_reduce(tableau, callback, valeur_initiale)
// Le callback reçoit DEUX arguments :
//   $total = l'accumulateur (commence à la valeur initiale)
//   $note  = l'élément courant du tableau

$somme = array_reduce(
    $notes,
    function(float $total, float $note): float {
        return $total + $note;
    },
    0.0   // valeur initiale de $total
);

// Voici ce qui se passe tour par tour :
// Tour 1 : $total = 0.0,  $note = 15.0  → retourne 15.0
// Tour 2 : $total = 15.0, $note = 12.0  → retourne 27.0
// Tour 3 : $total = 27.0, $note = 18.0  → retourne 45.0
// Tour 4 : $total = 45.0, $note = 9.0   → retourne 54.0
// Résultat final : 54.0

echo $somme;   // 54.0
```

## **`usort` — reçoit deux éléments à comparer**

```php
$eleves = [
    ["nom" => "Josh",   "note" => 5.3],
    ["nom" => "Martin", "note" => 18.3],
    ["nom" => "Sabine", "note" => 8.9],
];

// usort passe DEUX éléments à ton callback
// Tu dois retourner :
//  -1 (ou négatif) → $a avant $b
//   0              → ordre indifférent
//   1 (ou positif) → $b avant $a

// Tri par note croissante
usort($eleves, function(array $a, array $b): int {
    return $a['note'] <=> $b['note'];
    //  Si $a < $b → retourne -1 → $a avant $b ✅ croissant
});

// Tri par note décroissante — on inverse $a et $b
usort($eleves, function(array $a, array $b): int {
    return $b['note'] <=> $a['note'];
    //  Si $b > $a → retourne 1 → $b avant $a ✅ décroissant
});
```

# PARTIE 4 — La différence clé entre anonyme et arrow function

C'est ici que beaucoup de développeurs se trompent.

## L'accès aux variables extérieures

```php
$tva = 0.21;
$remise = 0.10;

// ❌ Arrow function — accède aux variables du scope parent AUTOMATIQUEMENT
$prixTTC = array_map(
    fn(float $prix): float => $prix * (1 + $tva) * (1 - $remise),
    $prix
);
// $tva et $remise sont accessibles directement ✅

// ⚠️ Fonction anonyme — N'accède PAS aux variables extérieures par défaut
$prixTTC = array_map(
    function(float $prix): float {
        return $prix * (1 + $tva) * (1 - $remise);  // ❌ $tva et $remise undefined !
    },
    $prix
);

// ✅ Fonction anonyme — il faut utiliser 'use' pour importer les variables
$prixTTC = array_map(
    function(float $prix) use ($tva, $remise): float {
        return $prix * (1 + $tva) * (1 - $remise);  // ✅
    },
    $prix
);
```

---

## Résumé visuel
```
┌─────────────────┬──────────────────────┬─────────────────────┐
│                 │  Fonction anonyme    │  Arrow function     │
├─────────────────┼──────────────────────┼─────────────────────┤
│ Syntaxe         │ function(...) {}     │ fn(...) => ...      │
│ Plusieurs lignes│ ✅ Oui               │ ❌ Non (une ligne)  │
│ Scope extérieur │ use($var) requis     │ Automatique         │
│ return explicite│ ✅ Obligatoire       │ ❌ Implicite        │
│ Usage idéal     │ Logique complexe     │ Logique simple      │
└─────────────────┴──────────────────────┴─────────────────────┘
```

# PARTIE 5 — Exemples progressifs complets

## Exemple 1 — `array_map` simple

```php
<?php
declare(strict_types=1);

$salaires = [2500.0, 3200.0, 4100.0, 1800.0];

// Appliquer une augmentation de 5%
$salairesAugmentes = array_map(
    fn(float $s): float => round($s * 1.05, 2),
    $salaires
);

var_dump($salairesAugmentes);
// [2625.0, 3360.0, 4305.0, 1890.0]
```

## Exemple 2 — `array_filter` avec condition complexe

```php
<?php
declare(strict_types=1);

$employes = [
    ["nom" => "Alice", "salaire" => 2500.0, "actif" => true],
    ["nom" => "Bob",   "salaire" => 4200.0, "actif" => true],
    ["nom" => "Carol", "salaire" => 1800.0, "actif" => false],
    ["nom" => "Dave",  "salaire" => 3100.0, "actif" => true],
];

$seuil = 3000.0;

// Garder les employés actifs avec un salaire > seuil
// use($seuil) nécessaire car fonction anonyme
$eligibles = array_values(array_filter(
    $employes,
    function(array $e) use ($seuil): bool {
        return $e['actif'] === true && $e['salaire'] > $seuil;
    }
));

// Avec arrow function — plus concis car $seuil accessible automatiquement
$eligibles = array_values(array_filter(
    $employes,
    fn(array $e): bool => $e['actif'] === true && $e['salaire'] > $seuil
));

var_dump($eligibles);
// Bob et Dave uniquement
```

## Exemple 3 — `array_reduce` pour construire un tableau

`array_reduce` ne sert pas qu'à calculer une somme — on peut aussi **construire un tableau** :

```php
<?php
declare(strict_types=1);

$employes = [
    ["nom" => "Alice", "departement" => "RH"],
    ["nom" => "Bob",   "departement" => "Dev"],
    ["nom" => "Carol", "departement" => "RH"],
    ["nom" => "Dave",  "departement" => "Dev"],
    ["nom" => "Eve",   "departement" => "Marketing"],
];

// Grouper les employés par département
$parDepartement = array_reduce(
    $employes,
    function(array $groupes, array $e): array {
        $dept = $e['departement'];

        // Si le département n'existe pas encore dans $groupes, on l'initialise
        if (!isset($groupes[$dept])) {
            $groupes[$dept] = [];
        }

        // On ajoute l'employé dans son département
        $groupes[$dept][] = $e['nom'];
        return $groupes;
    },
    []   // tableau vide comme valeur initiale
);

var_dump($parDepartement);
// [
//   "RH"        => ["Alice", "Carol"],
//   "Dev"       => ["Bob", "Dave"],
//   "Marketing" => ["Eve"]
// ]
```

## Exemple 4 — Chaîner `filter` + `map` + `reduce`

```php
<?php
declare(strict_types=1);

$commandes = [
    ["id" => 1, "montant" => 150.0,  "payee" => true],
    ["id" => 2, "montant" => 320.0,  "payee" => false],
    ["id" => 3, "montant" => 89.0,   "payee" => true],
    ["id" => 4, "montant" => 540.0,  "payee" => true],
    ["id" => 5, "montant" => 210.0,  "payee" => false],
];

$tva = 0.21;

// Étape 1 — Garder uniquement les commandes payées
$payees = array_values(array_filter(
    $commandes,
    fn(array $c): bool => $c['payee'] === true
));

// Étape 2 — Calculer le montant TTC de chaque commande
$avecTTC = array_map(
    fn(array $c): array => [...$c, 'montantTTC' => round($c['montant'] * (1 + $tva), 2)],
    $payees
);

// Étape 3 — Calculer le total TTC de toutes les commandes payées
$totalTTC = array_reduce(
    $avecTTC,
    fn(float $total, array $c): float => $total + $c['montantTTC'],
    0.0
);

echo number_format($totalTTC, 2, ',', ' ') . " €";
// (150 + 89 + 540) * 1.21 = 942.39 €
```

# 🎯 Exercice ciblé callbacks

Voici un tableau de **factures** :

```php
$factures = [
    ["id" => 1, "client" => "Dupont",  "montant" => 1200.0, "payee" => false, "jours_retard" => 30],
    ["id" => 2, "client" => "Martin",  "montant" => 450.0,  "payee" => true,  "jours_retard" => 0],
    ["id" => 3, "client" => "Bernard", "montant" => 3200.0, "payee" => false, "jours_retard" => 15],
    ["id" => 4, "client" => "Leroy",   "montant" => 890.0,  "payee" => true,  "jours_retard" => 0],
    ["id" => 5, "client" => "Moreau",  "montant" => 670.0,  "payee" => false, "jours_retard" => 5],
];

$tauxPenalite = 0.005; // 0.5% par jour de retard
$tva = 0.21;
```

Avec **uniquement** `array_filter`, `array_map`, `array_reduce` et `usort` :

1. **Filtrer** les factures non payées avec `array_filter`
2. **Enrichir** avec `array_map` chaque facture non payée :
    - `penalite` : `montant * tauxPenalite * jours_retard`
    - `montant_total` : `montant + penalite`
    - `montant_ttc` : `montant_total * (1 + tva)`
3. **Trier** par `montant_ttc` décroissant avec `usort`
4. **Calculer** le total TTC à récupérer avec `array_reduce`
5. Afficher avec `var_dump()` le tableau final et le total

⚠️ Contrainte : utilise `use()` dans au moins une fonction anonyme et une arrow function dans le même exercice.

Correction :

### ÉTAPE 1 — `array_filter` sur les factures non payées

```php
$factures = [
    ["id" => 1, "client" => "Dupont",  "montant" => 1200.0, "payee" => false, "jours_retard" => 30],
    ["id" => 2, "client" => "Martin",  "montant" => 450.0,  "payee" => true,  "jours_retard" => 0],
    ["id" => 3, "client" => "Bernard", "montant" => 3200.0, "payee" => false, "jours_retard" => 15],
    ["id" => 4, "client" => "Leroy",   "montant" => 890.0,  "payee" => true,  "jours_retard" => 0],
    ["id" => 5, "client" => "Moreau",  "montant" => 670.0,  "payee" => false, "jours_retard" => 5],
];

// array_filter parcourt le tableau
// Pour chaque facture $f, le callback doit retourner true ou false
// true  = on garde la facture
// false = on la supprime

$nonPayees = array_values(array_filter(
    $factures,
    fn(array $f): bool => $f['payee'] === false
));

// Résultat — seulement Dupont, Bernard, Moreau
```

Ici on utilise une **arrow function** `fn` car la condition est simple — une seule ligne, pas de variable extérieure à importer.

### ÉTAPE 2 — `array_map` pour enrichir

C'est ici qu'on a besoin de `use()` car on utilise `$tauxPenalite` et `$tva` qui sont des variables **extérieures** à la fonction anonyme.

```php
$tauxPenalite = 0.005;
$tva          = 0.21;

// On doit utiliser une fonction anonyme (pas arrow function)
// car la logique est sur plusieurs lignes
// Et on doit importer $tauxPenalite et $tva avec use()

$facturesEnrichies = array_map(
    function(array $f) use ($tauxPenalite, $tva): array {

        // Calcul de la pénalité
        $f['penalite'] = $f['montant'] * $tauxPenalite * $f['jours_retard'];

        // Montant + pénalité
        $f['montant_total'] = $f['montant'] + $f['penalite'];

        // Montant total avec TVA
        $f['montant_ttc'] = $f['montant_total'] * (1 + $tva);

        return $f;   // ← OBLIGATOIRE avec array_map
    },
    $nonPayees
);
```

**Pourquoi `use()` ici et pas avec l'arrow function du filtre ?**

```php
// Arrow function — $tauxPenalite accessible AUTOMATIQUEMENT
// MAIS on ne peut pas écrire plusieurs lignes avec une arrow function
fn(array $f): array => ... // Une seule expression possible

// Fonction anonyme — plusieurs lignes possibles
// MAIS les variables extérieures ne sont PAS accessibles automatiquement
// Il faut les importer explicitement avec use()
function(array $f) use ($tauxPenalite, $tva): array {
    // Ici $tauxPenalite et $tva sont disponibles
    ...
}
```

### **ÉTAPE 3 — `usort` pour trier**

```php
// usort modifie le tableau directement (pas de retour)
// Le callback reçoit deux éléments $a et $b à comparer
// L'opérateur <=> retourne -1, 0 ou 1

// Tri décroissant — $b avant $a
usort(
    $facturesEnrichies,
    fn(array $a, array $b): int => $b['montant_ttc'] <=> $a['montant_ttc']
);

// Ce qui se passe concrètement :
// PHP compare Bernard (4620.77€) vs Dupont (1635.66€)
// $b['montant_ttc'] > $a['montant_ttc'] → retourne 1 → $b avant $a ✅
```

### **ÉTAPE 4 — `array_reduce` pour le total**

```php
// array_reduce parcourt le tableau
// $total = accumulateur (commence à 0.0)
// $f     = facture courante

$totalTTC = array_reduce(
    $facturesEnrichies,
    fn(float $total, array $f): float => $total + $f['montant_ttc'],
    0.0   // valeur initiale
);

// Tour par tour :
// Tour 1 : $total = 0.0      + Bernard  4620.77 → 4620.77
// Tour 2 : $total = 4620.77  + Dupont   1635.66 → 6256.43
// Tour 3 : $total = 6256.43  + Moreau    963.68 → 7220.11
```

# Le schéma mental à retenir
```
┌─────────────────────────────────────────────────────────┐
│  CALLBACK = une fonction que tu passes à une fonction   │
├─────────────────────────────────────────────────────────┤
│  array_filter  → callback retourne bool                 │
│                  true = garder / false = supprimer      │
│                                                         │
│  array_map     → callback retourne l'élément modifié   │
│                  return OBLIGATOIRE                     │
│                                                         │
│  array_reduce  → callback reçoit ($total, $element)    │
│                  retourne le nouvel accumulateur        │
│                                                         │
│  usort         → callback reçoit ($a, $b)              │
│                  retourne -1 / 0 / 1 via <=>           │
├─────────────────────────────────────────────────────────┤
│  Arrow fn()    → une ligne, scope parent automatique   │
│  function(){}  → plusieurs lignes, use() obligatoire   │
└─────────────────────────────────────────────────────────┘
```
